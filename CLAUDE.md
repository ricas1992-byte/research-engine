# מאגר התובנות — הקשר פרויקט

## מהות הפרויקט

מאגר התובנות הוא סביבת חשיבה דיגיטלית עבור יותם ריקאס, מורה לפסנתר וחוקר פדגוגיה הפועל מחוץ למערכת הקונסרבטוריונים/אקדמיות הקלאסית. המערכת משמשת כ"מוח שני" בעבודה מול ד"ר זכריה פלאבין (מנטור, כ-8 שנות דיאלוג) על פרויקטי מחקר פילוסופיים-פדגוגיים.

## מבנה היררכי

```
שאלת מחקר → קטגוריות → חקירות → תובנות → תוצר סופי
```

כל רובד הוא יישות עצמאית במערכת עם קישורים דו-כיווניים לרבדים הסמוכים.

## מונחי יסוד (terminology)

- **תובנה:** יחידת הידע הבסיסית. משפט אחד או כמה משפטים שמבטאים הבנה.
- **חקירה:** תהליך עיבוד סביב תובנה אחת או צומת תובנות.
- **תוצר סופי:** מסמך שמסנתז חקירות בשלות לכדי טיעון או מפה.
- **אימון (deliberate training)** ≠ **תרגול (mechanical repetition)** — הבחנה יסודית בפדגוגיה של יותם.
- **פעמה (embodied pulse)** — מונח מרכזי.

## סטטוסים של תובנה

- **גולמי:** נקלטה, עוד לא נחקרה
- **מעובד:** בחקירה או עברה חקירה ראשונית
- **מוכן:** ניסוח מחודד, מוכנה להיכלל בתוצר סופי

## צירים (axes)

6 צירים תמטיים — כל תובנה משויכת לפחות לאחד:
1. כללי
2. תיאורטי
3. ביצועי
4. פסיכולוגי
5. ממסדי
6. פדגוגי

## פרויקט פעיל נוכחי: פרויקט הפריפריה v2

**שאלת מחקר:**
> איך יוצרים איזון בין דיגיטציה (טכניקה ודיוק בפרטים הדידקטיים) לפרשנות אומנותית (מה עומד מאחורי התווים)?

**מסגרת מושגית:**
- **פריפריה** = מקומות ללא כוח תבוני לפרשנות אומנותית עצמאית
- **מטרופולין** = מקומות בעלי צבר של אנשים עם כוח תבוני המאזן בין פרשנות אומנותית לדיגיטציה
- **דיגיטציה** = טכניקה ודיוק בפרטים הדידקטיים
- **פרשנות אומנותית** = מה שעומד מאחורי התווים

## פרויקט מאורכב: פרויקט הפריפריה v1

שאלת מחקר מקורית של זכריה: *"מה האדם שבפנים לא רואה כאן?"*
מאורכב — לא למחוק, לא להעביר נתונים. ניתן לקריאה דרך פילטר ארכיון.

## הנחיות עבודה

### שפה וכיווניות
- כל המסמכים והממשקים בעברית
- RTL תמיד: `dir="rtl"`, `text-align: right`, `bidirectional: true` במסמכי docx
- לטקסטים מעורבים עברית/אנגלית: TextRuns נפרדים

### סגנון קוד
- React + Vite + Zustand (stack נוכחי)
- TypeScript כשאפשר
- קומפוננטים פונקציונליים
- שמות משתנים באנגלית, מחרוזות UI בעברית

### workflow
- Direct execution — ללא approval checkpoints
- Commit אחרי כל שינוי משמעותי
- הודעות commit בעברית
- לפני push: `npm run lint` + `npm run build` חייבים לעבור

### כללי זהב
- אל תמחק נתוני משתמש לעולם. ארכוב בלבד.
- אל תציע קטגוריות/תובנות/חקירות מדעתך — המשתמש מגדיר את המבנה
- אל תשנה שאלת מחקר בלי אישור מפורש
- שמור קישורים דו-כיווניים תקינים תמיד

## ארכיטקטורה טכנית

- **Frontend:** React 18 + Vite + TypeScript + Tailwind CSS
- **State:** Zustand (main store = Supabase-synced; project store = localStorage-persisted)
- **Backend:** Supabase (PostgreSQL + Row Level Security per user_id)
- **Auth:** Supabase Auth (email + password)

## ניהול פרויקטים

פרויקטים מנוהלים ב-`src/data/projectStore.ts` — Zustand store עם localStorage persistence.
שיוך קטגוריות לפרויקט נשמר ב-`categoryProjectMap` (categoryId → projectId).
קטגוריות ללא שיוך = שייכות ל-`periphery-v1` (ברירת מחדל לנתונים ישנים).

## קבצים חשובים

- `src/data/store.ts` — Zustand store ראשי (Supabase-synced)
- `src/data/projectStore.ts` — ניהול פרויקטים (localStorage)
- `src/types/index.ts` — כל הטיפוסים
- `src/lib/api/` — שכבת API ל-Supabase
- `supabase/migrations/` — קבצי migration
- `.claude/skills/` — Skills ייעודיים לפרויקט
- `CLAUDE.md` — הקובץ הזה

## היסטוריית פרויקטים

- **periphery-v1** (מאורכב) — שאלת זכריה המקורית, מיפוי עמדת הפריפריה
- **periphery-v2** (פעיל) — שאלת האיזון בין דיגיטציה לפרשנות אומנותית
